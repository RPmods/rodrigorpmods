Updates / Historial Mejoras importantes
  • v2.3 — Base estable: se consolidó la interfaz competitiva, el draft local, bloqueos, selecciones, audios y resumen final.
  • v2.4 — Se añadió el modo online con salas por código, líder espectador, sincronización de picks, bans, turnos, mapa y estado completo.
  • v2.5 — Se corrigieron errores de sincronización de turnos, temporizador, reconexión, selección aleatoria y bloqueo de acciones por rol.
  • v2.6 — Se añadieron eventos sincronizados de fase, avisos grandes por equipo, selección de capitanes desde el lobby y cierre de sala por el líder.
  • v2.7 — Se mejoró la sincronización de sonidos, voces, preselección, animaciones, ruleta automática y selección automática de mapa.
  • v2.8 — Se corrigió el lobby online, el flujo de nombre al unirse, el botón de desconectar, nombres aleatorios y ajustes visuales del HUD.
  • v2.9 — Se añadió precarga inicial de recursos, indicador de carga, precarga secundaria y ocultamiento suave de HUD durante anuncios de fase.
  • v3.0 — Se añadió portugués, sistema de narración por idioma, lectura de bot para idiomas sin voces grabadas, historial completo y soporte visual para más mapas.
  • v3.1 — Se añadió mini lobby de configuración local, tamaños 2v2/3v3/4v4/5v5, opción para saltar la fase de bloqueos y sincronización de esta configuración en salas online clásicas.
  • v3.1 Hotfix — Se desbloqueó la selección de Modo avanzado para 2v2/3v3/4v4/5v5, se hizo el video de fondo independiente por usuario y la precarga crítica ahora aparece justo después del aviso F11.
  • v3.1.2 HOTFIX (Security) — Se rotó la RPmods Services Web API Key usada en GitHub Pages, se retiró la clave anterior expuesta y se reforzó la documentación de seguridad para uso público.
  • v3.1.3 HOTFIX (UI) — Se corrigieron los selectores desplegables con fondo/texto inconsistente en Idioma, Audio de narración y mini-lobby online; además se homogeneizó el estilo visual de los controles del lobby de crear sala.
  • v3.1.4 HOTFIX (Online Presence) — Se desactivó la restauración automática silenciosa de salas online al abrir la web para evitar escrituras en RPmods Services solo por cargar la página; la reconexión queda como acción explícita mediante código de sala.
  • v3.1.5 HOTFIX (Env Config) — Se movió la configuración web de RPmods Services a un archivo generado desde variables de entorno/Repository Secrets para GitHub Pages; se retiró la API key directa del código fuente público.
  • v3.1.6 HOTFIX (GitHub Pages Env Path) — Se corrigió el paquete para GitHub Pages con la estructura correcta: workflow en la raíz del repositorio y generación automática del archivo de entorno de RPmods Services.
  • v3.2.0 Modo Avanzado — Se añadió asignación avanzada por jugador para 2v2, 3v3, 4v4 y 5v5; el líder puede jugar o quedar como espectador, los nombres se rellenan desde usuarios conectados, los bans avanzados se reparten entre capitán/subcapitán y los picks son individuales.
  • v3.2.1 Hotfix (Advanced Voice/Text) — Se corrigieron los textos del modo avanzado para distinguir entre tu turno y el turno de tu equipo; se añadieron claves de voz para las nuevas líneas de bloqueo/selección de laminantes y soporte i18n para los idiomas disponibles.
  • v3.2.2 Testing Bots — Se añadió un apartado de Testing Bots para que el host pueda rellenar capitanes o slots vacantes con bots; los bots pueden bloquear y seleccionar automáticamente en modo clásico y avanzado para facilitar pruebas online.
  • v3.2.3 Hotfix (Testing Bots) — Se corrigió el fallo al rellenar slots vacantes con bots en modo avanzado; ahora el guardado de slots evita conflictos de rutas en RPmods Services.
  • v3.2.4 Hotfix (Online Finish + Host Name) — Se restauró el anuncio/voz online de finalización del draft usando voice_finish_draft y se añadió un apartado para que el líder/host pueda escribir su nombre antes de asignarse como capitán o jugador.
  • v3.2.5 Hotfix (Bot Thinking Delay) — Los bots ahora simulan mejor un turno real: tardan entre 3 y 6 segundos, hacen varias preselecciones aleatorias de laminantes válidos y luego confirman el bloqueo o la selección.
  • v3.2.6 Ready Check + Visual Turn UI — Se redujo y reubicó el cuadro de turno online, se ocultó el panel antiguo detrás del aviso, se añadió sistema de listo antes del draft con cajas por jugador, transición con Loading.mp4 y bots con nombres aleatorios desde player_names_config.js.
  • v3.2.7 Hotfix Visual + Selección Aleatoria — Se ocultó el mini panel superior redundante sin tocar el panel inferior del draft, se mejoró el aviso final de “el sistema draft ha concluido”, se añadió botón de selección aleatoria con narración, se corrigió la lógica de facciones válidas para los bots y se refinó el ready check online con un estilo inspirado en la referencia enviada.
  • v3.2.8 Hotfix (Random Button + Ready Polish) — Se reposicionó el botón de selección aleatoria dentro del panel inferior para no flotar ni romper el selector, y se pulió la entrada visual del ready check con animación de paneles/cajas más cercana a la referencia.
  • v3.2.9 Hotfix (UI Alignment + Ready Timeout + Map Polish) — Se reajustaron el aviso de turno y el panel inferior, se corrigieron las tarjetas del ready check con nombres dentro de cajas uniformes, se añadió timeout de 30 segundos para listos, se mejoró la ruleta de mapas con borde dorado/brillo y se añadió botón para que jugadores puedan retirarse al terminar.
  • v3.2.10 Hotfix (Turn Banner + Ready Countdown) — Se corrigió la posición/tamaño del banner de turno propio para que tape menos al personaje, se restauró el turno enemigo como aviso grande centrado, se ajustó el panel inferior para que BANEAR/SELECCIONAR y SELECCIÓN ALEATORIA tengan el mismo ancho, y el contador de listo ahora se actualiza localmente sin quedarse congelado.
  • v3.2.11 Hotfix (Panel Buttons Alignment) — Se igualó el ancho de BANEAR/SELECCIONAR y SELECCIÓN ALEATORIA, se movió la caja de información del personaje más a la izquierda y se ajustó el panel inferior para que los controles queden compactos y alineados.
  • v3.2.12 Hotfix (Equal Action Buttons) — Se igualó el ancho real de BANEAR/SELECCIONAR y SELECCIÓN ALEATORIA, se fijó la columna de acciones y se movió la caja de personaje para que el panel inferior se vea alineado.
  • v3.2.13 Hotfix (Deep Equal Buttons) — Se corrigió a profundidad el ancho real de BANEAR/SELECCIONAR y SELECCIÓN ALEATORIA anulando reglas antiguas heredadas; también se redujo el tamaño del botón/texto de espera en el ready check.
  • v3.2.14 Hotfix (Selected Button + Map Voice) — Se corrigió que BANEAR/SELECCIONAR se ensanchara al seleccionar personaje usando reglas más específicas para el estado visible/.ban; también se corrigió la voz de selección de mapa para que no se reproduzca dos veces.
  • v3.2.15 Update (Online Sync + Animation Performance + RPmods Services) — Se redujeron las escrituras y reconstrucciones del modo online, las ruletas ahora se reproducen desde una sola secuencia sincronizada, se eliminó el listener duplicado de sala, se suavizó la recuperación del video de fondo y se consolidaron las animaciones de bloqueo/selección para reducir filtros y renderizados innecesarios. Las referencias visibles del servicio online ahora usan el nombre RPmods Services.
  • v3.2.16 Update (Modern Draft Animations) — Se modernizaron las confirmaciones de BAN y selección con impactos visuales diferenciados, barridos de luz y transiciones optimizadas; las casillas de jugadores ahora conservan su DOM, muestran estados vacío/activo/previsualización/seleccionado y reproducen una revelación moderna del retrato y los datos sin reiniciar la animación con cada sincronización.
  • v3.2.17 Update (Integrated Online Notices) — Se reemplazaron los cuadros nativos del navegador por avisos integrados en la interfaz. La indisponibilidad de RPmods Services ahora aparece como una tarjeta animada junto a los controles online, con reintento y cierre; los demás errores y mensajes usan notificaciones no bloqueantes dentro de la página.
  • v3.2.18 Update (Online Connection Gate) — Antes de crear una sala, abrir el formulario de acceso o confirmar una unión, la aplicación muestra “Comprobando conexión a RPmods Services”, verifica el estado real del servicio y solo continúa si hay conexión. Si falla, conserva la acción pendiente y muestra RECONECTAR para reanudarla automáticamente.
  • v3.2.19 Update (Silent Multi-Intro Precache) — Se añadieron cinco parejas sincronizadas de video y música para la intro. Al aceptar el aviso principal se elige una pareja sin repetir la anterior, se precarga silenciosamente con prioridad alta y se mantiene preparada en memoria; las demás parejas se almacenan gradualmente en caché cuando el navegador está libre.
  • v3.2.20 Hotfix (Multi-Intro Selection) — Se corrigió el fallback que confundía una precarga lenta con un archivo inválido. Las intros 2–5 ahora mantienen su selección mientras cargan y solo vuelven a la intro original cuando el MP4 o MP3 genera un error real.
  • v3.2.21 Update (Dedicated Intro Media Structure) — Las cinco parejas de intro ahora se organizan en carpetas independientes dentro de media/intro-menu/intro-01 hasta intro-05. Cada carpeta contiene video.mp4 y music.mp3; las rutas antiguas permanecen como fallback temporal para facilitar la migración.
  • v3.2.22 Update (Intro Selection Settings) — Se añadió un apartado en Configuración para activar la selección aleatoria de las cinco intros o desactivarla y elegir manualmente un conjunto. La preferencia se guarda localmente y utiliza el mismo sistema de precarga silenciosa.
  • v3.2.23 Update (Named Intro Sets) — Los cinco conjuntos de intro ahora muestran sus nombres reales en Configuración y en el estado del modo manual, manteniendo intactas sus rutas, parejas de video/música y precarga silenciosa.
  • v3.2.26 Hotfix (Stable Flow Restore) — Se restauró la base funcional estable del modo online, el inicio del draft local y los botones de retorno/cancelación. También se retiraron los cambios experimentales que podían cortar listeners, flujo de turnos y navegación.
